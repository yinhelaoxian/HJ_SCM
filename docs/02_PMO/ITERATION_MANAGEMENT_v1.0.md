## 1. Sprint 规划模板

### 1.1 Sprint 定义模板

| 字段 | 说明 |
|------|------|
| Sprint 编号 | Sprint X |
| 起止日期 | YYYY-MM-DD 至 YYYY-MM-DD |
| 迭代目标 | 本 Sprint 要达成的核心目标 |
| 交付件 | 本 Sprint 要交付的具体成果 |
| 团队容量 | 人天数、可用工时 |
| 风险项 | 识别到的风险及应对措施 |

### 1.2 Sprint 规划会议议程

1. **产品经理** - 介绍 Sprint 目标（15分钟）
2. **业务专家** - 补充业务验收标准（10分钟）
3. **高级架构师** - 技术可行性评估（15分钟）
4. **团队拆解** - 任务拆解与估算（30分钟）
5. **确认交付标准** - 明确验收条件（10分钟）

## 2. 每日站会规范

### 2.1 站会流程

| 时间 | 内容 | 负责人 |
|------|------|--------|
| 5分钟 | 昨日完成 | 每人 |
| 5分钟 | 今日计划 | 每人 |
| 5分钟 | 阻塞问题 | 主持人 |
| 5分钟 | 风险升级 | 主持人 |

### 2.2 站会检查清单

- [ ] 所有成员已就位
- [ ] 任务看板已更新
- [ ] 阻塞问题已记录
- [ ] 需要升级的问题已标记

## 4. 文档更新同步机制

### 4.1 文档变更触发规则

| 变更类型 | 触发条件 | 同步要求 |
|----------|----------|----------|
| PRD 变更 | 需求范围调整 | 同步更新技术文档 |
| API 变更 | 接口参数调整 | 同步更新前端代码 |
| 状态机变更 | 状态/事件调整 | 同步更新测试用例 |
| 设计变更 | 组件/交互调整 | 同步更新 DESIGN_SYSTEM |

### 4.2 文档版本管理

| 文档类型 | 版本格式 | 更新频率 |
|----------|----------|----------|
| PRD | v1.0, v2.0 | 每次迭代 |
| 技术规范 | v1.0, v1.1 | 重大变更 |
| API 文档 | v1.0, v2.0 | 每次发布 |
| 组件规范 | v1.0, v1.1 | 组件变更 |
